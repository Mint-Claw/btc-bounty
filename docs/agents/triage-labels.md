# Triage labels

Canonical local labels for BTCBOUNTY work:

- `needs-triage`: maintainer/operator review needed.
- `needs-info`: blocked on missing human/system context.
- `ready-for-agent`: autonomous agent can implement from the issue alone.
- `ready-for-human`: needs a human product/security/deployment decision.
- `wontfix`: intentionally not planned.

Use AFK/HITL in issue bodies even when using local markdown instead of a tracker with labels.
