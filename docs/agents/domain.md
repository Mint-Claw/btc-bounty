# Domain docs

BTCBOUNTY uses a single-context layout:

- `CONTEXT.md` at repo root for product/domain language.
- `docs/adr/` for hard-to-reverse or surprising decisions.
- `docs/issues/` for local vertical-slice issues until GitHub Issues are adopted.

Agents must read `CONTEXT.md` before changing UI copy, API names, payment state, Nostr event handling, or public-alpha docs.
