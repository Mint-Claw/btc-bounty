# Agent issue tracker

Primary tracker: local markdown issue files under `docs/issues/` until GitHub Issues are explicitly adopted for this repo.

Issue files should be vertical slices: each issue must deliver a narrow, demoable path through all relevant layers rather than a horizontal backend/frontend split.

Use this naming pattern:

`docs/issues/YYYYMMDD-<slug>.md`

Each issue should include:

- Type: AFK or HITL
- Goal
- Acceptance criteria
- Verification commands
- Receipt expected at completion
- Blockers
